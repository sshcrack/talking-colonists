## Critical Bugs

### C1. NPE via `assert` in client-side rendering path
**File:** `McTalkingClient.java:96-98`
**Issue:** `assert minecraft.player != null` is bypassed at runtime (assertions disabled by default). If `Minecraft.getInstance().player` is null (world unload during render), NPE at `entity.isInvisibleTo()`.
**Fix:** Replace `assert` with `if (minecraft.player == null) return;`

### C2. NPE in `EndConversationAction.execute()`
**File:** `EndConversationAction.java:20-23`
**Issue:** `ConversationManager.getClientForEntity()` returns null (simple `Map.get()`), but `client.endConversationWhenPossible()` is called without null-check.
**Fix:** Guard with `if (client == null) return errorResponse;`

### C3. NPE in `ConversationManager.endConversation()`
**File:** `ConversationManager.java:388`
**Issue:** `activeEntity.remove(playerId)` returns null if player not mapped; `releaseSlot(entity)` then calls `entity.getUUID()` with null.
**Fix:** Guard with `if (entity != null) releaseSlot(entity);`

## High-Severity Bugs

### H1. Unsynchronized reads on `HashMap` in `ConversationManager`
**File:** `ConversationManager.java:58-67`
**Issue:** `clients`, `activeEntity`, `citizenToPlayer` are plain `HashMap` with synchronized writes but unsynchronized reads (`getClientForEntity`, `getActiveEntityForPlayer`, `getPlayerForEntity`). `HashMap` is not safe for concurrent read/write.
**Fix:** Synchronize read methods too, or use `ConcurrentHashMap`.

### H2. Static `Random` seed mutation breaks randomness
**File:** `AvailableAI.java:22,37`
**Issue:** A single static `Random` instance has its seed mutated on every `getRandomVoice` call. Affects all concurrent uses of `random` including mumbling chance rolls.
**Fix:** Create per-call `Random` or use `ThreadLocalRandom` with deterministic per-UUID seed.

### H3. First status change never produces AI prompt
**File:** `CitizenDataMixin.java:54-66`
**Issue:** On first `setVisibleStatus` call (`lastStatus == null`), the method sets `lastStatus` then immediately returns because `lastStatus.equals(status)` is now true. The actual status update (lines 76-83) is skipped for the very first change.
**Fix:** Move `setLastStatus` to after the guard, or restructure logic.

### H4. Non-volatile `isGenerating` across threads
**File:** `PregenerationTaskService.java:23`
**Issue:** `isGenerating` is read/written across threads without `volatile` or `AtomicBoolean`. Writes may not be visible to other threads.
**Fix:** Use `AtomicBoolean` or `volatile`.

### H5. NPE from unchecked `getCitizenData()`
**File:** `GeminiWsClient.java:109-112`
**Issue:** `entity.getCitizenData()` called without null-check before `isFemale()`/`isChild()`.
**Fix:** Add null-check or throw `IllegalArgumentException`.

### H6. Non-thread-safe `onCloseActions` list
**File:** `GeminiWsClient.java:91`
**Issue:** `onCloseActions` is a plain `ArrayList` written from WS thread (`addOnCloseAction`) and read from server/WS thread (`close()`). Race can cause `ConcurrentModificationException`.
**Fix:** Use `CopyOnWriteArrayList` or `Collections.synchronizedList()`.

### H7. Lost exception stack trace in error logging
**File:** `LiveConversationWsClient.java:163-164` (varies by version)
**Issue:** Only `ex.getMessage()` is logged, not the exception itself. Stack trace is lost.
**Fix:** Pass exception to logger: `LOGGER.error("msg", ex)`

### H8. Memory leak: memory generators never removed on completion
**File:** `PlayerConversationMemoryGenerator.java:165`, `CitizenMemoryGenerator.java:154`
**Issue:** Threads only remove themselves from `activeGenerators` on `interrupt()` or uncaught exceptions. On normal `run()` completion, they remain referenced forever.
**Fix:** Use `try-finally` in `run()` to remove from list.

### H9. Inverted logic in `onServerStart`
**File:** `ServerEventHandler.java:68-76`
**Issue:** Returns early when API key IS set, printing "key not set" error only when key is present. The intent is to warn when key is missing.
**Fix:** Invert condition.

### H10. `LeaveColonyAction` NPE on null `getCitizenData()`
**File:** `LeaveColonyAction.java:73-74,87`
**Issue:** `data = citizen.getCitizenData()` not null-checked before `data.serializeNBT()`.
**Fix:** Add null-check and return error.

### H11. `RecordRelationshipChange` always returns empty response
**File:** `RecordRelationshipChange.java:86-87`
**Issue:** `obj` is constructed but never populated with `"success": true` — the method returns whatever was in `obj` (empty if no error).
**Fix:** Add `obj.addProperty("success", true)` before return.

### H12. API key exposure risk
**File:** `GeminiWsClient.java:102`, various
**Issue:** API key passed as plain `String` through constructor, visible in heap dumps.
**Fix:** Consider credential wrapper or masking in logs.
